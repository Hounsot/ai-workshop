# Готовимся к воркшопу по работе с нейросетями

Этот чеклист поможет заранее подготовить компьютер, доступы и рабочее окружение для участия в воркшопе. Основной инструмент воркшопа - **Cursor AI**. Дополнительно могут демонстрироваться другие AI-инструменты, например Claude, ChatGPT или Claude Code.

Пожалуйста, пройдите подготовку **до начала воркшопа**. На установку, оплату, настройку VPN и проверку доступов может уйти больше времени, чем кажется.

# Выжимка

- Скачайте и установите `Cursor`.
- Войдите в аккаунт `Cursor` и проверьте, что подписка или доступ к AI-функциям активны.
- Подготовьте `VPN` на личном устройстве и не устанавливайте его на рабочий компьютер.
- Проверьте, что Cursor отвечает вам и в чате внутри редактора, и в режиме `Agent`.
- Зарегистрируйтесь на `GitHub`, если аккаунта еще нет.
- Заранее поставьте все технические инструменты для практики: [дополнительные требования](#дополнительные-требования).
- По возможности подготовьте второй монитор, планшет или другое устройство для просмотра трансляции.

## Что понадобится

### Компьютер

Подойдет macOS, Windows или Linux. Желательно использовать ноутбук или компьютер, на котором вы сможете:

- устанавливать приложения;
- авторизоваться в сторонних сервисах;
- открывать внешние сайты;
- запускать Cursor AI;
- запускать VPN;
- работать с файлами проекта.

Если у вас корпоративный компьютер без прав администратора, заранее проверьте, сможете ли вы установить Cursor и сопутствующие инструменты. Если установка заблокирована, обратитесь в IT до воркшопа.

### Права администратора

Права администратора могут понадобиться для:

- установки Cursor;
- установки Git, Node.js и других инструментов;
- обновления системных компонентов;
- настройки разрешений безопасности на macOS или Windows.

Если прав администратора нет, лучше решить этот вопрос заранее. Во время воркшопа на это может не хватить времени.

### Второй экран

Второй монитор не обязателен, но сильно упростит участие. Идеальный вариант:

- на основном экране открыт Cursor;
- на втором экране открыта трансляция или материалы воркшопа.

Если второго монитора нет, можно использовать планшет, другой ноутбук или телефон для просмотра трансляции.

## Основной инструмент: Cursor AI

Cursor AI - это редактор кода с AI-ассистентом. На воркшопе он будет использоваться как основная среда для практических заданий.

### Установка

1. Перейдите на сайт Cursor: [https://www.cursor.com/](https://www.cursor.com/)
2. Скачайте версию для вашей операционной системы.
3. Установите приложение.
4. Запустите Cursor и войдите в аккаунт.
5. Проверьте, что AI-чат внутри Cursor работает.

Если вы уже используете VS Code, Cursor будет выглядеть знакомо: это отдельный редактор со схожим интерфейсом. При этом в Cursor есть и отдельное окно `Agent`, которое визуально очень похоже на интерфейс ChatGPT и Claude, поэтому начать работать с ним обычно легко даже тем, кто раньше пользовался AI-чатами только в браузере.

### Ваш первый проект в Cursor

После установки Cursor откройте любой проект и попробуйте базовые AI-функции. Это поможет понять, что все настроено правильно и вы готовы к практической части воркшопа.

#### Как открыть проект в Cursor

1. Откройте Cursor.
2. Выберите `File > Open Folder`.
3. Укажите папку с вашим проектом.

Если у вас пока нет рабочего проекта, можно заранее создать отдельную папку для практики на воркшопе.

#### Как начать работать с Agent

Agent - это AI-ассистент внутри Cursor. Он умеет:

- искать информацию по проекту;
- читать и анализировать файлы;
- предлагать и вносить изменения в код;
- запускать команды в терминале.

Чтобы открыть панель Agent в режиме редактора кода:

- macOS: `Cmd + I`
- Windows / Linux: `Ctrl + I`

После этого можно написать запрос в чат. Например:

```text
Привет! Кратко объясни, чем Cursor отличается от обычного VS Code.
```

После отправки запроса Agent начнет изучать проект, предложит изменения и покажет, что именно он сделал.

#### Как проверять изменения Agent

Изменения применяются по мере работы Agent. Обязательно просматривайте их в diff view и отклоняйте все, что вам не подходит.

Важно не воспринимать предложения AI как автоматические "правильные ответы": Agent помогает ускорить работу, но финальное решение остается за вами.

#### Как дать Agent больше контекста

Обычно Agent сам ищет нужные файлы по проекту. Но если вы хотите направить его точнее, используйте `@` в поле ввода.

Примеры:

- `@auth.ts`
- `@src/components/`

Это особенно полезно, если проект большой и вы хотите сразу указать нужный файл, папку или область работы.

### Аккаунт и подписка

Для комфортной работы на воркшопе лучше заранее проверить лимиты и доступность AI-функций. Бесплатного тарифа будет недостаточно, особенно если во время практики будет много запросов к модели.

Рекомендуемый вариант - заранее оформить подходящую подписку Cursor:

- Pro;
- Ultra, если вам нужен расширенный лимит.

## Доступность сервисов из России

Мы будем проводить воркшоп из России, поэтому часть AI-сервисов может быть недоступна напрямую или работать нестабильно. Это может касаться Cursor AI, Claude, ChatGPT, Claude Code и других сервисов.

Перед воркшопом проверьте:

- открывается ли сайт Cursor;
- проходит ли авторизация;
- работает ли AI-чат в Cursor;
- доступна ли оплата выбранного тарифа;
- стабильно ли работает сервис из вашей сети.

Лучше сделать проверку за несколько дней до воркшопа, а не за 10 минут до старта.

Важно: **не устанавливайте VPN на рабочий компьютер**.

### Если у вас macOS

На Mac при установке `Node.js`, `npm`-пакетов или других инструментов командной строки может понадобиться установить Apple `Command Line Tools`.

Обычно это нужно, если:

- терминал показывает ошибку про отсутствующие developer tools;
- система просит установить command line tools;
- не ставятся зависимости, которым нужны системные утилиты для сборки.

Установить их можно командой:

```bash
xcode-select --install
```

После этого macOS откроет системное окно установки. Лучше сделать это заранее, до воркшопа, потому что установка может занять время.

## Безопасность и данные

Во время работы с AI-инструментами не используйте:

- пароли;
- токены доступа;
- приватные ключи;
- персональные данные клиентов или сотрудников;
- коммерчески чувствительные материалы;
- внутренние документы компании без разрешения.

Если нужно протестировать сценарий, используйте обезличенные или тестовые данные.

## Проверка готовности перед воркшопом

За день до воркшопа откройте Cursor и проверьте:

1. Приложение запускается.
2. Вы вошли в аккаунт.
3. AI-чат отвечает на простой запрос.
4. Подписка активна.
5. VPN работает.
6. Есть чуть-чуть места на диске для материалов воркшопа.
7. Второй экран или устройство для просмотра трансляции готовы.

Пример тестового запроса в Cursor:

```text
Привет! Кратко объясни, чем Cursor отличается от обычного VS Code.
```

Если Cursor ответил, базовая настройка прошла успешно.

## Что делать, если что-то не работает

Если не получается установить Cursor:

- проверьте права администратора;
- попробуйте скачать установщик еще раз;
- обратитесь к [Матвею](https://t.me/hounsout).

Если не работает авторизация или AI-чат:

- проверьте интернет;
- попробуйте другую сеть;
- проверьте доступность сервиса через VPN;
- убедитесь, что подписка или лимиты аккаунта активны.

Если не получается оплатить подписку:

- попробуйте МТС Оплату;
- попробуйте Plati.Market или другой проверенный способ;

## Минимальная готовность для участия

Минимально нужно:

- установленный Cursor;

- доступ к AI-функциям Cursor;
